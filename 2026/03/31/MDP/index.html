关于 MDP 的一些思考

  • 状态，动作，<状态，动作> -> 状态，奖赏，价值，策略，动作价值
  • 状态一定是有限的
  • 未来状态的概率只与当前状态有关，与历史状态无关
  • 动作选择与当前状态和选择策略有关
  • 每一个 <状态，动作> -> 状态 都会带来一个奖赏
  • 价值 V(s) = max(Reward + V(s’)) ，注意当前状态的新价值和自己的旧价值无关，是价值更新前其他状态的价值决定的
  • 价值迭代的本质是 「动态规划」，从终点（宝箱）反向传播最优价值 ，所以是会收敛的
    • reward 已知
    • state 和 action 离散
  • 策略迭代分为两步：策略评估和策略提升
    • 策略评估是根据当前策略直接算出每个状态呃期望价值
    • 策略提升是计算当前状态的最好Action，更新策略
    • 不停重复，直至策略不再改变
  • 价值迭代可看做简化版的策略迭代，相当于两步融合到一步走了
